# Криптосистема на QC-MDPC кодах над $F_p$ и $F_4$

В этом репозитории содержится оптимизированная реализация криптосистемы на QC-MDPC кодах над полем $F_4$, а также реализация над полями $F_p$, где p - простое число.

Реализация над полем $F_4$ -- библиотека на C, исходники находятся в `src/gf4x`, пример использования в `src/gf4x_benchmark.

Реализация над полем $F_p$ -- библиотека на C++, исходники находятся в `src/gfpx`. Она не оптимизирована и скорее является референсом.

Во многом эта криптосистема похожа на криптосистему [BIKE](https://bikesuite.org/), которая также является криптосистемой на QC-MDPC кодах, только над бинарными полями ($F_2$). Достигнутая производительность в случае $F_4$ возможна благодаря открытой реализации BIKE: https://github.com/awslabs/bike-kem, в частности циклические свертки над $F_2$ были полностью взяты из этой реализации, и алгоритм обращения многочлена был адаптирован под $F_4$.

Узнать подробнее о QC-MDPC кодах и нужной теории можно в `paper/main.pdf`.

## Результаты

У криптосистемы есть три основных операции: генерация публичного и приватного ключей, кодирование и декодирование. В таблице приведено сравнения производительности по этим трем операциям.

| Алгоритм       | N     | Операция | Время выполнения (мс) |
|---------------:|:------|:---------|----------------------:|
| BIKE Level 3   | 24659 | Keygen   | 12.717               |
|                |       | Encaps   | 0.730                |
|                |       | Decaps   | 12.419               |
| GF4X           | 24646 | Keygen   | 12.587               |
|                |       | Encaps   | 0.721                |
|                |       | Decaps   | 4.607                |

Время работы Decaps существенно отличается из-за разных декодеров, используемых в Decaps. В BIKE используется декодер, выполняющий 4 итерации, в то время как декодер в этой работе выполняет всего одну итерацию.

Результатом работы стало то, что получилось показать практическую производительность у криптосистем над $F_4$. Результат ничем не уступает бинарному BIKE, и в то же время криптосистемы над $F_4$ могут быть более устойчивы к атакам за счет чуть более сложной структуры чем $F_2$.
